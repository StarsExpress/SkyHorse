1944. Visible People

Number of Visible People in a Queue​

非常好玩的一道难题 我原本其实也被数学公式唬了

什么叫做看得见➡️右边的人​

题目页有定义：总人数nnn的队列

假设有索引iii和jjj 其中0≤i<j<n0 \leq i < j < n0≤i<j<n

若第iii和第jjj人 他们中间全部的人都比这俩矮

那么第iii人才能看见第jjj人

因此题目写了一道数学公式作为判断依据：

min(heights[i],heights[j])>maxi<k<j(heights[k])min(heights[i], heights[j]) > max_{i < k < j}(heights[k])min(heights[i],heights[j])>maxi<k<j​(heights[k])

咀嚼数学公式后能察觉​

给定某索引0≤i<j<n−10 \leq i < j < n - 10≤i<j<n−1

要是第j+1j + 1j+1人没有比第jjj人高

那么第iii人和第j+1j + 1j+1人一定不满足前段数学公式

因此第iii人看不见第j+1j + 1j+1人

说到这 是不是就感觉到 单调性又来啰😀

结合由右向左遍历时 第j+1j + 1j+1人迭代到后

紧接著是第jjj人被迭代 第iii人还要过阵子才轮到

自然能把第j+1j + 1j+1人从可能被左方人看的选项划掉

于是这选项不就是个后进先出的栈嘛

具体操作​

弹栈神通🍿​

每次第jjj人被迭代时 就把栈顶的人和第jjj人比较

只要栈顶的人比第jjj人矮 就把栈顶的人弹出

同时要给 第jjj人能向右看见的人数加一

因为如此的弹法下 栈顶的人和第jjj人之间的所有人都比他们俩矮

栈弹到何时为止？弹到终于 栈顶的人不比第jjj人矮

小心细节㊙️️​

弹到终于 栈顶的人不比第jjj人矮

进入了一个微妙的细节 要特别注意

(1). 首先 到这节骨眼 栈顶人肯定能被第jjj人瞧见

不管这位栈顶人是比第jjj人高 还是说一样高

第jjj人能向右看见的人数自然加一

毕竟开头说的数学公式能办到嘛

(2). 巧妙的来啦 栈顶人倘若比第jjj人高

那第jjj人左方的人 还有机会就看栈顶人

因为开头说的数学公式还有机会做到呀

但是这个栈顶人要是和第jjj人一样高

那么第jjj人左方的所有人都看不见栈顶人

因为开头说的数学公式做不到啦

得赶快弹走这位与第jjj人一样高的栈顶人

简而言之 这边的操作是

两个分支情况都要给第jjj人向右的能见度加一

但只有在栈顶人和第jjj人同高时 才要弹栈顶

千萬別弄混淆啦

搞定弹栈后 第jjj人再进栈顶

#include <stack>
#include <vector>
using namespace std;

vector<int> countVisibilities(vector<int>& heights) // LeetCode Q.1944.
{
    stack<int> stack; // A stack of heights.

    vector<int> visibilities(heights.size(), 0);

    for (int idx = heights.size() - 1; idx >= 0; idx--) {
        int height = heights[idx];

        while (!stack.empty() && stack.top() < height) {
            stack.pop();
            visibilities[idx]++;
        }

        if (!stack.empty()) {
            visibilities[idx]++;

            if (stack.top() == height)
                stack.pop();
        }

        stack.push(height);
    }

    return visibilities;
}

def count_visibilities(heights: list[int]) -> list[int]:  # LeetCode Q.1944.
    stack = []
    visibilities = [0] * len(heights)

    for idx in range(len(heights) - 1, -1, -1):  # Backward iteration.
        height = heights[idx]
        
        while stack and stack[-1] < height:  # People on my left side can't see them.
            stack.pop(-1)
            visibilities[idx] += 1  # I can see unblocked shorter people.

        if stack:  # I can always see my next non-shorter person (if exists).
            visibilities[idx] += 1

            if stack[-1] == height:  # People on my left side can't see this person.
                stack.pop(-1)
        
        stack.append(height)

    return visibilities

空间复杂度：O(n)O(n)O(n) 有可能大家都比右边人矮 都被锁在栈内
